# Contributing to CogniVerse

Thanks for your interest in contributing. We welcome issues, bug reports, and pull requests.

Getting started
1. Fork the repo and create a feature branch: `git checkout -b feature/your-change`.
2. Install dependencies in `backend/` and (if needed) `frontend/`.
3. Run the project locally and add tests for new features.

PR Guidelines
- Keep changes small and focused.
- Follow the existing code style (vanilla JS, Express, Mongoose models).
- Include tests where applicable and update docs.
- Link your PR to an issue, or create an issue describing the change first.

Communication
- Be polite and follow the `CODE_OF_CONDUCT.md`.
